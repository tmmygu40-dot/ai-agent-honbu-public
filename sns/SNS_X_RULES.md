## 1. 投稿文テンプレート（必須形・全候補この形）

```text
【今日のチェック】

一言フック。

アプリの説明文。

アプリ名
↓　↓　↓

https://nekopoke.jp/s/<slug>/
```

- 1行目: `【今日のチェック】` 固定
- 空行 1
- 2行目: 一言フック（敬語なし・あるある寄り）
- 空行 1
- 3行目: アプリの短い説明（1〜3行）
- 空行 1
- 4行目: アプリ名（コピペしやすい単独行）
- 5行目: `↓　↓　↓` 固定（全角矢印3個 + 全角スペース区切り）
- 空行 1
- 最終行: short_url

---

## 2. URLルール

- **short_url 優先**: `https://nekopoke.jp/s/<slug>/` を本文に使う
- /s/ 未作成の場合: 投稿文では `https://nekopoke.jp/<URL-encoded-app-name>/` （実在 dir の直URL）を使う・本文に長くなっても**実在URLを優先**
- slug は提案段階なら `slug_status: "proposed"` と明記（dry-run JSON）
- **404候補は投稿不可** — 投入前に URL 実在確認必須
- queue 投入時の 404 検出ロジックは `_reports/x_unexplored_short_url_dryrun_*.md` を参照

## 3. フック基本ルール

### 文体
- **敬語なし**: 「です / ます / でしょう / ください / なります / あります」は使わない
- 友達に言うような短い一言
- ただし**煽りすぎない**
- ネコポケらしく「あるある」「ちょい便利」「ちょい笑い」寄り

### 良いフック例（採用基準）
- カードは増える。記憶は増えない。
- あとで整理するカード、だいたい一生あと。
- ジム行く前の準備、地味に毎回ぬける。
- 冷蔵庫の中身、覚えてるつもりでだいたい忘れる。
- 防災グッズ、あるだけで安心して中身を見てない。
- メルカリ出品、値段決めで毎回止まりがち。
- 退去費用、最後のボス感ある。（SNS_RULES.md 既存例）
- 給料、入った瞬間どこかへ旅立つ説。（SNS_RULES.md 既存例）

## 4. 禁止フック

| 禁止パターン | 例 |
|---|---|
| 「フック未決定」プレースホルダ | "フック未決定" / "(TBD)" |
| 「決まりませんでした」系 | "決まりませんでした" |
| 「〜問題。」だけで終わる | "値上げ問題。" |
| タイトルの言い換えだけ | "○○記録アプリ" → "○○を記録する" |
| 意味が薄い一文 | "便利です" / "整理したい" |
| 過度な煽り / 不安強調 | "今すぐ確認しないと損" |
| 医療・健康・投資・相場で**断定** | "絶対痩せる" / "必ず儲かる" |
| 禁止ワード（SNS_RULES.md 継承）| 「絶対」「必ず」「保証」「儲かる」「誰でも稼げる」 |

## 5. 🔥 ハマった失敗ルール（2026-06-17 hook polish 反省）

### 失敗事案
400件 polish で **20カテゴリ別ローテーション** hookを全件に当てた結果、以下のズレが発生:

| 例 | カテゴリ | ローテで当たった hook | ズレの理由 |
|---|---|---|---|
| ポケカポイント還元交換比率比較 | ポケカ | 「前日の準備、毎回忘れて朝慌てる」 | 還元/ポイント系に大会準備系hookが混入 |
| マタニティ食事制限ガイド | 子育て | 「学校プリント、迷子になりがち」 | マタニティに学校行事系hookが混入（ライフステージ早い）|
| 自然売り買い見送り | TCG売買判断 | 「デッキ箱の中身、なんか足りない予感がする」 | 売買判断系に持ち物系hookが混入 |

### 再発防止ルール（必須）

#### R1: カテゴリ別ローテーション hook を**そのまま全件に当てない**
- 同カテゴリでも**サブテーマ**で hook を分ける
- 機械的ローテは ❌
- app_name の意味で再選定するロジック必須

#### R2: app_name / title の意味と hook がズレるなら **不採用**
- カテゴリだけで決めず、app_name キーワードで最終フィルタ
- マッチが弱ければ「手動 hook 作成 candidate」フラグを立てる
- 自動置換できない場合は `manual_review: true` で人間レビューへ送る

#### R3: 同カテゴリでも別トーンに分けるサブテーマ
TCG カテゴリ内でも以下は**別 hook プール**を持つ:
- 大会準備 / プレリリ系
- 相場 / 値段 / 売買判断系
- 収納 / ファイル / 保管系
- 開封 / パック / BOX系
- トレード / 交換系
- 鑑定（PSA/BGS/CGC）系
- 旧裏 / コレ系
- スリーブ / ローダー系

#### R4: LIFE系の**繊細な扱い**
以下は別 hook プールで扱う:
- マタニティ / 産前産後 → 子育て一般 hook と分ける
- 学校 / 学校行事 → 育児一般 hook と分ける
- 通院 / 服薬 → ダイエット食事 hook と混ぜない
- ペット → 食事系/運動系から分離（ペット専用 hook）
- 家計 / シミュレーター → 家事 hook と分ける

#### R5: hook polish では **category より title 優先**
- 機械的に category で hook 選ぶより、title の語幹で hook 選ぶ
- 例: 「マタニティ」「学校」「通院」「ペット」「家計」「相場」「開封」「鑑定」「旧裏」「スリーブ」「保管」「収納」「トレード」「大会」を見る

## 7. ジャンル配分ルール

### 200件 dry-run 生成時
- **TCG だけで 200件を埋めない**（在庫多様性のため）
- 1セット 200件は 15〜20ジャンルでミックス
- 各ジャンルは最低 5件、最大 14件 を目安
- TCGバッチを別途出す場合は 200件単位で OK

### 避けるジャンル（量を増やさない）
- 占い / MBTI / 性格診断 / 相性診断
- 推し活（過去候補に多数あるため）
- 脳トレ / 一般クイズ
- 心理テスト / 血液型 / 星座
- ※ゼロでなくてもいいが**今回は増やさない**

### 推奨配分（生活混合200件の場合）
- ジム/筋トレ/運動: 14
- ダイエット/体重/食事: 14
- 生活便利/家事/掃除/片付け: 14
- 買い物/節約/家計: 14
- 仕事/業務/会議: 14
- 子育て/家族/学校: 13
- ペット/犬猫: 13
- 防災/備蓄/点検: 13

## 8. dry-run 運用フロー（投入前ゲート）

```
[1] 200件 dry-run 生成
    ↓ (sns/_reports/x_*_candidates_200_dryrun_YYYYMMDD.{md,json})
[2] 既存重複チェック
    ↓ queue (sns_queue.json) + registry (SNS_DEDUP_REGISTRY.json)
    ↓ + 過去 dry-run (sns/_reports/)
    ↓ + 公開アプリ dir + workspace dir
[3] URL実在確認
    ↓ verified_url が public repo の実 dir or /s/ slug にあるか
[4] hook polish（必要なら）
    ↓ §3 / §4 / §5 適用
[5] 兄貴 / BOB 確認
    ↓ 目視レビュー・気になる候補は手動修正
[6] queue 投入 (兄貴 GO 後のみ)
    ↓ public/sns/sns_queue.json append (status=draft)
    ↓ public/sns/SNS_DEDUP_REGISTRY.json entries に追加
[7] commit & push (兄貴 GO 後のみ・明示ファイルのみ)
```

### ゲート遵守事項
- **dry-run なしで queue 投入禁止**
- **commit/push は兄貴の明示 GO 後のみ**
- `git add . / -A / -u / -f` 絶対禁止
- 既存 M / 既存 ?? を勝手に stage しない
- public/backup repo の無断変更禁止
- `factory_state_v2.json` / baseline JSON は stage 禁止

## 10. 候補エントリの最小フィールド

dry-run JSON の各 entry に以下を必ず含める:

```json
{
  "id": "YYYYMMDD-<TAG>-NNN",
  "type": "x_idea",
  "category": "<ジャンル名>",
  "title": "<タイトル / app_name>",
  "hook": "<§3 ルール準拠の一言>",
  "description": "<説明 1-3行>",
  "app_name": "<実 dir 名 or 提案 app 名>",
  "slug_candidate": "<英数字 slug>",
  "slug_status": "existing" | "proposed",
  "short_url_candidate": "https://nekopoke.jp/s/<slug>/",
  "verified_url": "<実在 URL>",
  "x_text": "<§1 テンプレ準拠>",
  "status": "dry_run",
  "created_at": "<ISO 8601>",
  "source": "<生成バッチ名>"
}
```
