# SNS RULES (X投稿文)

## 基本方針
- X投稿文は「面白おかしいけど怪しくない」文体を基本にする。
- 真面目すぎる説明文はタイムラインで流れやすい。
- 1行目はクスッとするフックにする。
- 2〜3行目で何が確認できるかを自然に説明する。
- 医療・保険・お金系は断定しない。
- 最後は「無料チェック👇」+ URL で統一する。

## 禁止ワード / NG表現
- 「絶対」「必ず」「保証」「儲かる」「誰でも稼げる」は使わない。
- 過剰に不安を煽らない。
- 下品・攻撃的・差別的な表現を使わない。

## フック例
- 退去費用、最後のボス感ある。
- 一人暮らし、財布のHP削られがち。
- 住民税、忘れた頃に現れるラスボス。
- 給料、入った瞬間どこかへ旅立つ説。
- 防災グッズ、買おうと思って3年経ってる。

## 追加ルール（2026-05-01 追記）
- X投稿文は長く説明しすぎない（短く読める構成を優先）。
- 1行目は「ちょい面白いフック」にする。
- 2〜3行目は短い疑問文で引っかける。
- 4行目で行動を促す。
- 注意書きは短く自然に入れる。
- 最後は URL で締める。
- 「無料チェック👇」は使ってもよいが、毎回固定で入れすぎない（広告感を避けるため文面に合わせて調整）。
- 断定や過剰煽りは避ける。

## 採用例
退去費用、最後にドンと来がち。

敷金で足りる？
原状回復ってどこまで見る？

あとで焦る前に、
ざっくり目安を無料チェック。
※断定ではなく、確認用です

https://nekopoke.jp/賃貸退去費用シミュレーター/

## 追加ルール（2026-05-01 追記2: 先頭空行）
- X表示では、アカウント名直下と本文1行目が近く見えるため、投稿文の先頭に空行を1つ入れる。
- 1行目は空行、2行目からフックを開始する。

## 基本構成（先頭空行版）
- 1行目: 空行
- 2行目: フック
- 空行
- 3〜5行目: 短いリズム文
- 空行
- 説明2行
- 注意書き
- 空行
- URL

## 採用例（先頭空行版）

一人暮らし、財布のHP削られがち。

家賃で削られ、
食費で削られ、
光熱費でトドメ刺されるやつ。

毎月いくら必要そうか、
ざっくり目安チェックできます。
※断定ではなく、確認用です

https://nekopoke.jp/一人暮らし実質生活費自動診断/

## 追加ルール（2026-05-01 追記3: 1行目ラベル方式）
- 先頭空行はX投稿画面側で詰まって見えることがあるため、運用を停止する。
- 投稿文の1行目は固定ラベル「【今日のチェック】」にする。
- 2行目を空行にし、3行目からフックを開始する。
- 以降の構成（リズム文、短い説明、注意書き、URL）は維持する。

## 追加ルール（2026-05-07: Xリンク・短縮URL）

### 問題
- 日本語パスだけの長い URL は、**X 上ではリンクが途中で切れたり、開くと 404 になることがある**。
- パーセントエンコードした URL は動きやすいが **長く見た目が悪い**。

### 運用
- **`url`**: ブラウザ用の実ページ（例: `https://nekopoke.jp/〇〇〇/`）。
- **`short_url`**: X 本文末尾だけに使う **短い英数字URL**（例: `https://nekopoke.jp/s/16bit-map/`）。
- **`s/{slug}/index.html`** を公開リポに置き、**実ページへリダイレクト**する。

### X向け末尾の鉄則
- パスに日本語が含まれるなら **`short_url` を付けてから投稿する**ことを原則にする。
- **`short_url` を追加したら本番で開いてリダイレクト先が正しいか確認してから投稿**する。
- **`sns_queue.json` はオブジェクト削除をせず、`short_url` は必要なオブジェクトだけに追加する**。

### X投稿並びの意識合わせ（新ダッシュの骨格）
- **悩み・あるある・使いたくなる一言** → **アプリ名** → **どう使えるか 1〜2 行** → **短縮URL** の順が読みやすい。（先頭ラベル方式と併用可。）
